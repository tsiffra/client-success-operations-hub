# Screenshots Checklist

Screenshots are intentionally not checked into the repo yet. Add them only when final captures are ready, and keep this checklist updated so the repository does not contain stale or misleading images.

Use this checklist before publishing the repository or adding the project to a resume/portfolio.

## Required Screenshots

- [ ] Dashboard
- [ ] Tickets
- [ ] Ticket Detail
- [ ] Customers
- [ ] Customer Detail
- [ ] Analytics
- [ ] About Project

## Capture Notes

- Use a desktop browser width so the sidebar and tables are visible.
- Capture the Dashboard with the Action Center and KPI cards visible.
- Capture a Ticket Detail page that shows the request, support answer, metadata panel, timeline, and internal notes.
- Capture a Customer Detail page that shows health score, ARR, renewal date, recommended next action, and recent tickets.
- Avoid showing local file paths or browser developer tools in final screenshots.
